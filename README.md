# Grubhunter
## Departures from the Norm
The activity PDFs are included in this repository so that I can open them exclusively, as opposed to the previews on Canvas.
As one may have noticed, the folder structure of this repository is slightly awry.
This is due to complications concerning the root folder.

If you follow the instructions in `grubHunterActivity1.pdf` to a tee, you will end up with a `code` folder as your root.
Following further, when you compose the docker container, a new directory will appear: `code/code/`.  
Strange, right?

So. To start, I made a root folder for the project, in which sits...
- the file `docker-compose.yml`
- the directory `.docker/grubhunter-backend/`
- some PDFs, but those are not important
- the autogenerated `code/` directory created by the composer

Once I reached step four in the `grubHunterActivity1.pdf`, the "root folder" effectively changed.
From this point on, for everything related to the NextJS "frontend", 
the "root folder" is the `code/` folder that was automatically generated by the composer.
This might be a bit tricky to follow, and for this I apologize.

## Having issues?
- **When you make the `volumes` for the `backend` service, trying to execute `docker compose up` may show that `"mongodb_data_container"` is undefined.**<br><br>
  At the bottom of your `docker-compose.yml`, ensure that you have a second `volumes` group with no indentation, and add that container to it like so:
  ```yml
  # <previous code, including services...>
  
  volumes:
      mongodb_data_container:
  ```
  Be sure not to forget that colon.<br><br>
- **Having trouble running those `docker exec ...` commands?**<br><br>
  In VSCode, you should have a terminal in which you run `docker compose up`.
  In the terminal widget/sub-window, look in the top right corner for a splitscreen icon:
  <img width="1468" height="353" alt="image" src="https://github.com/user-attachments/assets/75294165-179c-49e8-85a9-7ce563778ea4" /><br><br>
  This will open a second terminal beside the first one.
  In this second terminal, you can run these `docker exec` commands.
  After you `cd` into the `code/` folder, it will look something like this:
  `docker exec -it "grubhunter-application-1" npx create-next-app`
